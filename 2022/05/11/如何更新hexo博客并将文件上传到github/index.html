如何更新hexo博客并将文件上传到github

yi. generate

生成静态文件。

选项 描述
-d, –deploy 文件生成后立即部署网站
-w, –watch 监视文件变动
-b, –bail 生成过程中如果发生任何未处理的异常则抛出异常
-f, –force 强制重新生成文件Hexo 引入了差分机制，如果 public 目录存在，那么 hexo g 只会重新生成改动的文件。使用该参数的效果接近 hexo clean && hexo generate
-c, –concurrency 最大同时生成文件的数量，默认无限制

该命令可以简写为：

er. deploy

部署网站。

参数 描述
-g, –generate 部署之前预先生成静态文件

该命令可以简写为：

san. 把Test文件夹下面的文件都添加进来

wu. 提交修改信息

“修改信息”里面换成你需要，如”新提交”

liu. 把本地仓库push到github上面

此步骤可能需要你输入帐号和密码

至此本地项目代码就成功上传到远程Github仓库了，在Github主页刷新就可以看见自己的代码
